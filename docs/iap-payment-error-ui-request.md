# 인앱결제(IAP) 지급 실패 UI 요청

> 디자이너 전달용. 화면별 카피(제목/본문/버튼) 초안.
> 범위: `TOSS_SERVER_VERIFICATION_FAILED`, `PRODUCT_NOT_GRANTED_BY_PARTNER` 두 에러코드 관련 화면만.

---

## 배경

MATE는 테스트(설문) 발행 시 토스 인앱결제로 결제해요. 결제 자체(토스 결제창)는 성공했는데, 우리 쪽 상품 지급/테스트 발행 처리만 실패하는 경우(`TOSS_SERVER_VERIFICATION_FAILED`, `PRODUCT_NOT_GRANTED_BY_PARTNER`)에 대한 화면이 없어요.

이 두 에러는 아래 두 시점에 각각 필요해요.

1. 결제 진행 중 에러가 발생한 그 순간
2. 앱을 다시 열었을 때, 지연됐던 결제 건이 백그라운드에서 자동으로 재처리되는 순간

---

## 1. 결제 진행 중 에러

```
제목  결제는 완료됐지만, 테스트가 아직 등록되지 않았어요
본문  결제 금액은 정상적으로 처리됐어요.
      테스트 등록만 일시적으로 실패했으니, 다시 시도하면 등록할 수 있어요.
      계속 실패한다면 아래 주문번호로 문의해주세요.

      주문번호  {orderId}
      오류코드  {TOSS_SERVER_VERIFICATION_FAILED | PRODUCT_NOT_GRANTED_BY_PARTNER}

버튼  [문의하기]  [등록 다시 시도]
```

- 두 에러코드 모두 "결제는 됐다"는 전제라 같은 카피로 처리
- "상품 지급"은 IAP 개발 용어라 유저 관점에서 안 와닿음 → 기존 결제완료 화면(`PaymentCompleteStep.tsx`)의 "테스트 등록" 표현에 맞춰 통일
- **[등록 다시 시도] 버튼**: 이미 알고 있는 `orderId`/`draftId`로 바로 재지급 로직(`restore` → 성공 시 `completeProductGrant`)을 호출. 앱 재접속 시 자동 복구와 같은 로직을 재사용하되, `getPendingOrders` 조회 단계만 생략

---

## 2. 앱 재실행 시 자동 재지급 처리

토스 가이드상, 지급 실패 건은 **앱을 다시 열었을 때 자동으로 재지급을 시도**해야 해요(`getPendingOrders` → 재지급 → `completeProductGrant`). 성공 시에만 아래 화면으로 안내.

```
제목  밀렸던 결제가 처리됐어요
본문  {테스트명} 결제가 정상 처리돼서 테스트가 등록됐어요.
버튼  [확인]
```

- **노출 시점/위치**: `AuthGuard` 로그인 완료 직후, 특정 화면에 종속되지 않는 **전역 오버레이(모달/바텀시트)**로 1회 노출. 처리된 주문은 `getPendingOrders` 목록에서 다음부터 안 나오므로, 별도 "이미 봤음" 플래그 없이도 재노출 안 됨
